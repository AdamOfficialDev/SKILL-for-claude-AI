## Hardcoded String Detection

### What to Flag

Any user-visible string that is embedded directly in source code instead of a translation key:

```javascript
// ❌ Hardcoded — will not translate
return <h1>Welcome back!</h1>;
return <p>Error: Invalid email address</p>;
toast.error("Something went wrong. Please try again.");
throw new Error("User not found");
placeholder="Enter your name"
aria-label="Close menu"

// ✅ Translation key
return <h1>{t('greeting.welcome_back')}</h1>;
return <p>{t('errors.invalid_email')}</p>;
toast.error(t('errors.generic_retry'));
```

**Exceptions — do NOT flag:**
- String identifiers (enum values, event names, CSS class names)
- Developer-facing error messages in non-UI code (logging, internal exceptions)
- Technical strings (URLs, CSS values, regex patterns)
- Strings that are guaranteed to be language-neutral (e.g. ISO codes)

## String Concatenation Anti-Patterns

Word order differs between languages. Concatenating strings breaks translations.

```javascript
// ❌ Concatenation — unnatural word order in many languages
const msg = "Hello, " + username + "! You have " + count + " messages.";
// In Japanese: 「田中様、メッセージが3件あります。」— different order entirely

// ✅ Template with named placeholders
const msg = t('inbox.greeting', { username, count });
// en: "Hello, {username}! You have {count} messages."
// ja: "{username}様、メッセージが{count}件あります。"
// de: "Hallo, {username}! Sie haben {count} Nachrichten."
```

## Plural Forms

Most i18n libraries handle plural categories (zero/one/two/few/many/other). Never use manual if/else for plurals.

```javascript
// ❌ Only handles English singular/plural
const label = count === 1 ? "item" : "items";
// Arabic has 6 plural forms; Russian has 3; Chinese has 1

// ✅ Use i18n plural rules
const label = t('cart.item_count', { count });
// en.json: { "item_count": { "one": "{{count}} item", "other": "{{count}} items" } }
// ar.json: { "item_count": { "zero": "لا توجد عناصر", "one": "عنصر واحد", "two": "عنصران", "few": "{{count}} عناصر", "many": "{{count}} عنصرًا", "other": "{{count}} عنصر" } }
```

## Locale-Insensitive Operations

### Date Formatting

```javascript
// ❌ Locale-insensitive date
new Date().toLocaleDateString()           // Uses browser default locale
new Date().toLocaleDateString('en-US')    // Hardcoded to US format: MM/DD/YYYY

// ✅ Locale-aware
new Date().toLocaleDateString(userLocale, {
    year: 'numeric', month: 'long', day: 'numeric'
})
// en-US: "January 15, 2024"
// de-DE: "15. Januar 2024"
// ja-JP: "2024年1月15日"

// ✅ Using i18n library
format(date, 'PPP', { locale: getDateFnsLocale(userLocale) })
```

### Number Formatting

```javascript
// ❌ Hardcoded decimal/thousands separator
const formatted = (1234567.89).toFixed(2);  // Always "1234567.89"

// ✅ Locale-aware
const formatted = new Intl.NumberFormat(userLocale).format(1234567.89);
// en-US: "1,234,567.89"
// de-DE: "1.234.567,89"
// fr-FR: "1 234 567,89"
```

### Currency

```javascript
// ❌ Hardcoded currency symbol and format
const price = "$" + (amount / 100).toFixed(2);

// ✅ Locale + currency aware
const price = new Intl.NumberFormat(userLocale, {
    style: 'currency',
    currency: currencyCode  // 'USD', 'EUR', 'IDR', 'JPY'
}).format(amount / 100);
// en-US + USD: "$12.50"
// de-DE + EUR: "12,50 €"
// id-ID + IDR: "Rp 12.500"
// ja-JP + JPY: "¥1,250"
```

### Sorting

```javascript
// ❌ JS default sort doesn't respect locale
['Ångström', 'Zebra', 'apple'].sort()
// en result may differ from sv-SE result (Å sorts after Z in Swedish)

// ✅ Locale-aware sort
['Ångström', 'Zebra', 'apple'].sort((a, b) => 
    a.localeCompare(b, userLocale, { sensitivity: 'base' })
)
```

### Timezone Assumptions

```javascript
// ❌ Assuming local timezone
const today = new Date().toISOString().split('T')[0];
// This gives different dates for users in different timezones at midnight

// ✅ Always work in UTC or explicitly handle timezone
const formatter = new Intl.DateTimeFormat('en-CA', {
    timeZone: userTimezone,
    year: 'numeric', month: '2-digit', day: '2-digit'
});
```

## RTL (Right-to-Left) Layout

Languages: Arabic, Hebrew, Farsi, Urdu, and others.

### HTML Direction

```html
<!-- ✅ Set direction on html element -->
<html lang="ar" dir="rtl">

<!-- For mixed content -->
<p dir="auto">Mixed content that auto-detects direction</p>
<span dir="ltr">URL: https://example.com</span>
```

### CSS RTL Issues

```css
/* ❌ Directional CSS properties don't flip in RTL */
.button {
    padding-left: 16px;
    margin-right: 8px;
    border-left: 2px solid blue;
    text-align: left;
}

/* ✅ Use logical properties — auto-flip in RTL */
.button {
    padding-inline-start: 16px;  /* left in LTR, right in RTL */
    margin-inline-end: 8px;      /* right in LTR, left in RTL */
    border-inline-start: 2px solid blue;
    text-align: start;           /* left in LTR, right in RTL */
}
```

### Icon Direction

```html
<!-- Icons that imply direction should flip in RTL -->
<!-- ❌ Arrow always points right -->
<span class="arrow-right">→</span>

<!-- ✅ Use CSS to flip directional icons -->
<span class="arrow-forward">→</span>

/* CSS */
[dir="rtl"] .arrow-forward {
    transform: scaleX(-1);
}
```

## Layout Issues with Translation

```css
/* ❌ Fixed width will clip longer translations */
.button { width: 120px; overflow: hidden; }
/* "Submit" = 6 chars, but "Absenden" (German) = 8, "Wyślij" (Polish) = 6 */

/* ✅ Allow content-driven width with a minimum */
.button { min-width: 120px; width: max-content; max-width: 100%; }
```

```css
/* ❌ Fixed-height containers that assume text won't wrap */
.card-title { height: 24px; overflow: hidden; }
/* Translated text may be 40% longer and wrap to 2 lines */

/* ✅ Use min-height */
.card-title { min-height: 24px; }
```

## i18n Framework Patterns

### react-i18next
```javascript
// Setup check — make sure i18n is properly initialized
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

// ❌ Missing fallback language
i18n.init({ lng: 'en', resources: { en: { translation: {} } } });

// ✅ With fallback and interpolation config
i18n.use(initReactI18next).init({
    lng: 'en',
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
    resources: { en: { translation: {} }, id: { translation: {} } }
});
```

### Python (Django / gettext)
```python
# ❌ String not marked for translation
error_message = "User not found"
template = f"Welcome, {user.name}!"

# ✅ Marked for translation
from django.utils.translation import gettext_lazy as _
error_message = _("User not found")

# ✅ With named placeholders for translators
from django.utils.translation import gettext as _
template = _("Welcome, %(name)s!") % {'name': user.name}
```
